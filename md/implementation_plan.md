# 制作 LepoProxy 桌面代理软件实施计划

目前你拥有的 `index.html` 只是一个精美的**静态网页界面**（前端）。要让它成为一个真正能翻墙的**桌面软件**，我们需要为其添加一个**后端系统**，并将其打包成桌面可执行文件（`.exe`）。

你提到了 **Tauri** 和 **Sing-Box**，这正是非常现代且高效的技术组合：
1. **Tauri**: 负责将你的网页打包成占用极小内存的桌面窗口，并提供与系统底层交互的能力。
2. **Sing-Box**: 真正的“网络代理核心”（内核），负责处理底层的网络流量、加密和规则分流。

由于你目前没有任何编程基础，也没有安装相关的开发环境，以下是从零开始的完整计划。**只需你同意该计划，并按照第一步完成环境安装，后续的所有代码编写、组合和调试都可以由我来为你完成。**

---

## 🛠️ 第一阶段：准备开发环境（需要你手动配合）

通过检测，你的电脑上目前没有安装开发所需的底层语言环境。为了能编译 Tauri 软件，我们需要安装以下工具。

你可以打开**命令提示符 (cmd)** 或 **PowerShell**，分别运行以下命令进行快速安装（如果网络不畅，可能需要自备全局代理）：

1. **安装 Node.js (前端运行环境)**:
   ```powershell
   winget install OpenJS.NodeJS
   ```
2. **安装 Rust (Tauri 后端语言环境)**:
   ```powershell
   winget install Rustlang.Rustup
   ```
   *(安装 Rust 时，可能会提示你安装 Visual Studio C++ Build Tools，请根据屏幕提示同意安装，这是编译 Windows 软件必须的 C++ 库)*

> [!IMPORTANT]  
> **环境配置是唯一必须由你手动完成的步骤。** 安装完成后，请**重启你的电脑**，然后在聊天中告诉我：“环境装好了，继续下一步”，我就会开始自动为你编写代码。

## 🚀 第二阶段：项目初始化与架构搭建（由我执行）

在你安装好环境后，我将执行以下操作：
1. **初始化 Tauri 架构**: 在 `d:\Project\LepoProxy` 目录下创建一个标准 Tauri 项目。
2. **迁移界面**: 将你现有的 `index.html` 嵌入到 Tauri 项目的 `src` 目录中。
3. **引入内核**: 下载最新版的 Windows `sing-box.exe` 内核，将其作为“附加程序 (Sidecar)” 嵌入到项目中。

---

## ⚙️ 第三阶段：后端核心逻辑开发 - Rust（由我执行）

我要在 Tauri 的后端层写代码，让你的网页不仅是“看起来”连接了，而是真正启动内核：
1. **进程管理**: 编写 Rust 代码来启动、停止和监控隐藏在后台的 `sing-box.exe` 进程。
2. **配置生成**: 当你在界面上切换“全局/直连/规则”或更换“节点”时，Rust 会动态生成一份 `config.json` 喂给 Sing-box 内核。
3. **TUN 网卡控制**: 实现需要管理员权限的 TUN 虚拟网卡接管，实现真正的全局代理。
4. **窗口控制**: 将目前的窗口设定为“无边框、无系统标题栏”，并实现右上角的“X”按钮隐藏到系统托盘，以及系统托盘的右键菜单。

---

## 🎨 第四阶段：前端交互对接 - JavaScript（由我执行）

我要修改你的 `index.html`，让界面按钮的点击真正触发后端的操作：
1. **打通数据流**: 将界面上的“默认节点”、“延迟(ping)”改为真实请求，调用 Rust 获取真实延迟并渲染。
2. **对接连接按钮**: 把现在点击按钮后的 `setTimeout` 假动画，改为等待 Rust 返回“Sing-box 启动成功”的真实状态。
3. **对接设置项**: 将“开机自启”、“TUN开关”等选项与系统底层设置挂钩。
